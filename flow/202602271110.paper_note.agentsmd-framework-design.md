# AGENTS.md ノート

## TL;DR
- 本論文は、`AGENTS.md` などのリポジトリ文脈ファイルが実際にタスク解決率を改善するかを実験評価した研究。
- 主結果は「LLM生成コンテキストファイルは平均的に成功率をわずかに下げ、コストを20%以上増やす」。
- 人手作成ファイルはLLM生成より有効だが、改善幅は小さく、追加コストは発生する。

## 背景/貢献
- 背景: `AGENTS.md`/`CLAUDE.md` は急速に普及（公開リポジトリで6万件超）している一方、効果検証が不足していた。
- 貢献1: 開発者が実際に文脈ファイルを置いているリポジトリから新規ベンチマーク **AGENT BENCH** を構築。
- 貢献2: SWE-bench Lite と AGENT BENCH の両方で、複数エージェント・複数モデルを横断評価。
- 貢献3: 成否だけでなくトレース解析により、探索行動・テスト実行・ツール使用の変化を分析。

## 用語説明
- コンテキストファイル: `AGENTS.md`/`CLAUDE.md` 等、リポジトリ固有の指示を与えるための文書。
- AGENT BENCH: 12リポジトリ・138インスタンスから成る、本論文で新規構築された評価ベンチマーク。
- NONE / LLM / HUMAN: 文脈ファイルなし、LLM生成ファイル、開発者作成ファイルの3条件。

## 方法/設定
- データセット:
  - SWE-bench Lite: 300タスク / 11人気Pythonリポジトリ。
  - AGENT BENCH: 138インスタンス / 12リポジトリ（5694 PR から抽出、平均75%の変更行カバレッジ）。
- 対象エージェント: Claude Code, Codex, Qwen Code（計4モデル設定）。
- 評価指標: success rate、steps、推論コスト。
- 比較条件:
  - NONE: 文脈ファイルなし。
  - LLM: エージェント推奨手順で自動生成した文脈ファイル。
  - HUMAN: 開発者がコミットした文脈ファイル（AGENT BENCHのみ）。

## 結果/制約
- 主要結果（LLM生成ファイル）:
  - 成功率: SWE-bench Liteで平均 -0.5pt、AGENT BENCHで平均 -2pt。
  - steps: 平均 +2.45（SWE-bench Lite）、+3.92（AGENT BENCH）。
  - コスト: 平均 +20%（SWE-bench Lite）、+23%（AGENT BENCH）。
- 主要結果（開発者作成ファイル）:
  - LLM生成よりは高性能で、NONE比でも多くの設定で改善（ただし例外あり）。
  - 一方で steps は平均 +3.34、コストは最大 +19% と増加。
- 行動分析:
  - 指示に書かれたツールは実際に強く追従される（例: `uv`、repo固有ツール）。
  - ただし「リポジトリ概要」としては有効性が限定的で、探索は増えるが解決率改善に直結しない。
- 制約:
  - Python中心・特定ベンチ条件での評価であり、言語/ドメイン一般化は未検証。
  - 成功率以外（保守性・セキュリティ・効率）の品質軸は今後の課題。

## 気づき/疑問
- 気づき: 「文脈を増やせば良くなる」は成立せず、冗長な要求は探索コストだけ増やしやすい。
- 気づき: 実務では、文脈ファイルは最小限の拘束条件（必須ツール、テスト実行方法など）に絞る方が合理的。
- 疑問: OSS以外の社内モノレポ/独自ツール環境では、人手文脈の価値がより大きくなるか。
- 疑問: 生成時点の静的文書より、タスク依存で動的に要約・抽出する方式の方が有効ではないか。
