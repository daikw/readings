# AGENTS.md ノート

## TL;DR
- 67件の公開AIエージェントフレームワークを体系分析し、`AGENTS.md` の記述品質と実運用性能の関係を定量化した研究。
- GitHubスター数よりも、明示的なタスク構造・具体例・機械可読ルールを持つドキュメントの方が性能を予測しやすい、という主張。
- フレームワーク比較の焦点が「実装差」だけでなく「運用プロンプト設計資産」へ移っている点が実務的に重要。

## 背景/貢献
- 背景: コーディングエージェントは急増しているが、フレームワーク設計の良し悪しを比較する標準が弱い。
- 貢献1: 67フレームワークを対象に、ドキュメントと設計パターンを横断比較するデータセットを提示。
- 貢献2: 自動抽出パイプラインを人手検証し、高い一致率（要旨ベースで 94%超）を報告。
- 貢献3: ドキュメント品質特徴とエージェント性能の関連を分析し、設計原則を提案。

## 用語説明
- AGENTS.md: エージェント運用ルールを自然言語で記述する、実装横断の設計ドキュメント層。
- フレームワーク設計品質: タスク分解、制約明示、例示、評価基準の記述充実度を指す。
- ドキュメント中心設計: コード実装より先に行動規範・I/O契約を固定する開発実務。

## 方法/設定
- 対象: GitHub上の公開AIエージェントフレームワーク 67件。
- 手順: ドキュメント解析により設計特徴を抽出し、公開ベンチマーク成績との関連を評価。
- 検証: 自動抽出結果を人手アノテーションで突合し、一貫性を確認。

## 結果/制約
- 結果: 人気指標（スター数）より、ドキュメントの明示性・構造性が性能と相関しやすい。
- 結果: 高性能系は、行動境界・失敗時分岐・出力フォーマット指示が具体的な傾向。
- 制約: 相関中心の分析であり、因果（記述改善が直接性能改善を生むか）は追加検証が必要。
- 制約: 公開プロジェクト中心なので、クローズド実運用との差分は残る。

## 気づき/疑問
- 気づき: 実務では「コード最適化」より「タスク仕様書と運用規約の整備」が先行投資として効く。
- 気づき: コーディングエージェント評価では、モデル比較に加えて `AGENTS.md` 品質監査を標準化すべき。
- 疑問: モデルサイズ/ツールチェーン差を統制したときにも同傾向が維持されるか。
- 疑問: 長期運用での仕様ドリフトをどう検知し、ドキュメント更新を半自動化するか。
