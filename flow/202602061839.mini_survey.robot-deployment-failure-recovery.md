# 軽量サーベイ

## スコープ/基準
- 実環境配備で問題になる失敗検知・診断・回復の設計に焦点。
- 2021-2025のロボット実機/運用寄り研究を優先し、手法と運用の接続を重視。

## 各論文サマリ
- Deploying Robots in Everyday Environments (arXiv:2206.12719) — 内省不足/故障対応の難しさを実運用課題として整理し、ブラックボックス/監視/遠隔実験の設計指針を提示。
- Reliable Robotic Task Execution in the Face of Anomalies (arXiv:2510.23121) — 視覚異常検知と三段階回復（停止→局所摂動→安全リセット）を統合し、実機2シナリオで成功率改善。
- Data-Driven Robot Fault Detection and Diagnosis Using Generative Models (arXiv:2311.13866) — RBMベースのSFDDで故障検知/診断を統合し、実機の物流ロボット故障でも精度改善を報告。
- A Multimodal Handover Failure Detection Dataset and Baselines (arXiv:2402.18319) — 手渡し失敗のマルチモーダルデータセットを提供し、ビデオと力覚/位置の有効性を示す。
- Enhancing Video-Based Robot Failure Detection Using Task Knowledge (arXiv:2508.18705) — タスク知識を用いた映像失敗検知でF1改善、推論時拡張の効果も提示。
- Robot Action Diagnosis and Experience Correction by Falsifying Parameterised Execution Models (arXiv:2105.09599) — 反証的診断と経験修正のループで失敗原因の同定と成功率改善を狙う。

## 比較/差分
- 運用設計（2206）→検知/回復（2510）→診断学習（2311/2105）→データ基盤（2402/2508）と、層が分化。
- 2510はオンライン回復まで踏み込む一方、2311/2105は診断の精度・説明性が主軸。
- 2402/2508は「失敗検知器の学習可能性」を上げるデータ/知識面の補強に位置づく。

## ギャップ
- 実運用の統一指標（復旧時間/安全性/説明性）と、VLA系モデルへの統合評価が不足。
- 診断→回復→再学習の閉ループを実機で継続評価する枠組みがまだ弱い。

## 次アクション
- 2510の回復ループを、2311/2105の診断信号で拡張する設計案を作る。
- 2402/2508のデータ/知識を使って、検知の偽陽性/偽陰性コストを定量化する。
- 2206の運用設計観点に照らして、ログ仕様/時間同期/保守手順の最低要件を整理。
