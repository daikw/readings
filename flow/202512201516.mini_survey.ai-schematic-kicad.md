# 軽量サーベイ

## スコープ/基準
- 目的: AI支援で回路図生成〜PCB配置配線までを自動化する方法を整理し、KiCadで実装可能な形に落とす。
- 対象: 実装を意識した論文（配置・配線・回路図生成）と、KiCad連携が可能なOSSツール。
- 除外: 画像認識や検査のみの研究、IC設計専用のVLSI配線のみの研究。

## 各論文サマリ
- An Efficient and High-Quality Circuit Schematic Generation Method based on Action Mask and Graph Neural Network — 回路グラフから回路図レイアウトを生成するGNN+行動マスク。部品配置と配線の順序を行動空間で制約し、可読性の高い図面を狙う。KiCadではSKiDL→netlist→図面配置の自動化パイプに組み込める。
- From Network to Artwork Automatic Schematic Diagram Generation — ネットワークリストから図面へ落とし込む自動生成の古典的手法。部品の配置規則・配線整形の設計指針が中心で、LLM前段のルールベースとして流用しやすい。
- Environment-Adaptable Printed-Circuit Board Positioning Using Deep Reinforcement Learning — DRLでPCB部品配置を最適化。制約（衝突/配線長/領域）を報酬で表現し、設計条件の変化に適応する配置戦略を示す。
- Reinforcement Learning-Based Placement Method for Printed Circuit Board — PCB配置の強化学習フレームワーク。ベンチマーク構築と学習設定が焦点で、KiCadのpcbnew APIで評価指標を実装しやすい。
- Security-oriented printed-circuit-board routing with deep reinforcement learning — セキュリティ制約を入れた配線最適化のDRL。配線禁止領域やトポロジ制約を学習に組み込む設計で、KiCad+FreeRoutingの前処理/後処理に適用可能（SSRN版とVLSI誌版がある）。
- A deep reinforcement learning framework for optimized dummy pad placement in PCB electroplating — 製造工程のためのダミーパッド配置最適化。直接の回路図生成ではないが、配置制約/報酬設計の参考になる。

## 実装/ツール（KiCad中心）
- pcb-designer-ai-agent — 要件→BOM→フットプリント→回路図→配置配線→Gerberまでのエージェント型パイプライン。KiCadバックエンドが主。
- kicad-mcp-python — KiCad公式IPC APIを使ったMCPサーバ。LLMがKiCadを直接操作できる実装の基盤。
- KiCAD-MCP-Server / kicad-mcp — MCPでKiCadを自然言語制御するための実装群。IPCバックエンドやツールスキーマが充実。
- kicad-sch-api — `.kicad_sch` を直接読書きするPython API。回路図生成の出力先として有用。
- SKiDL — コードで回路を記述しnetlistを生成。LLMの出力を安全に構造化する入口として使える。
- KiKit — 製造出力/パネル化などの自動化。自動設計の後段の定型作業を吸収。
- FreeRouting — KiCadと連携可能なオートルータ。配線の実験/比較に使いやすい。
- EagilinsED_PCB-Design-Agent — Altium向けだが、LLMがレイアウト生成/レビューまで行う実装例として参考になる。
- GEN-AI-Based-timer-board-PCB — LLM支援で回路図/レイアウトを進めた実例。自動生成ではないが実運用のヒントになる。

## 比較/差分
- 回路図生成はGNN/ルールベース、PCB配置配線はDRLが主流で、データ表現（グラフ/グリッド/トポロジ）が分断している。
- 実装レベルでは「ネットリスト→回路図レイアウト→部品配置→配線」の各段階で最適化指標が異なり、統合評価が未成熟。
- KiCad連携は論文内で直接の実装記述が薄いが、IPC API/MCP/ファイルAPIが整っており後付け実装は可能。
- 公開実装は「LLMがKiCadを操作できる土台」が中心で、学習済みモデルによる自動配置/配線の完成度は限定的。

## ギャップ
- KiCad向けの公開ベンチマーク/評価スイートが不足し、再現性が低い。
- 回路図生成とPCB配置配線の一貫最適化（人間の読みやすさ+電気特性）が未整理。
- LLMを回路図生成に直接使う研究が少なく、データセット整備（部品記号/配線規約）が必要。
- KiCad IPC APIを使った実験の比較例が少なく、同じタスクでのツール間比較が難しい。

## 次アクション
- KiCadパイプライン案: LLMで要件→BOM→SKiDLでnetlist→GNN/ルールで回路図配置→pcbnew APIで配置→FreeRoutingで配線→kicad-cliでGerber生成。
- 最初の実装は「回路図生成」だけに絞り、GNN/ルールの入出力をKiCad netlist/sexprに固定する。
- `pcb-designer-ai-agent` をベースに、回路図生成ステップの出力をKiCadに接続する薄いアダプタを作る。
- 評価指標: 配線交差数、配線長、シート可読性（部品間距離/整列/ラベリング）を定義して比較可能にする。
- MCPサーバ群（kicad-mcp-python等）を使い、LLMの操作ログを保存して再現可能な実験手順に落とす。
